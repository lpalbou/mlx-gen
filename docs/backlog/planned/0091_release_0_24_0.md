## Smoke evidence (recorded 2026-07-23, TI2V-5B q8, 384x384, 9 frames, 2 steps, seed 42)

- (a) Same-seed output identity vs 0.23.1: PASSED in the honest form. The
  0.23.1 release commit (worktree `1e3258b`, identical locked deps: torch
  2.9.1 / transformers 5.0.0) and the full 0.24.0 tree produced 9/9
  bitwise-identical frames on raw YUV planes. This simultaneously re-proves
  the video-writer color-tag change preserves coded pixels (old writer vs new
  writer, same bits). A FIRST comparison against the PyPI 0.23.1 install in a
  different dependency set (torch 2.8.0 / transformers 5.8.0) DIFFERED (mean
  luma delta ~10): root-caused to UMT5 kernel numerics across
  torch/transformers versions, not to mlx-gen code — pinned by a probe showing
  numpy-vs-torch tokenization and encoding are bitwise identical within one
  environment (ids, masks, embeds), and by a step-level bisect (step1 == full
  tree == 0.23.1-same-env, all bitwise). Embedding hosts should expect
  same-seed output shifts when THEIR torch/transformers versions change; that
  is upstream-kernel behavior, not an mlx-gen contract.
- (b) Disk-cache hit vs miss: bitwise identical (raw YUV, 9/9 frames). Hit
  runs skip the UMT5 load entirely (cache-hit end-to-end run: ~27 s vs ~4 min
  miss on a contended GPU).
- (c) `--no-validate-health`: exit 0, output bitwise identical to the default
  run, sidecar records `health_check: "skipped"`; `--json-events` `start`
  event carries resolved `width`/`height` (observed live).
- (d) `MFLUX_RUN_LOCAL_WAN_PARITY=1` local parity suite: 6/6 passed
  (Wan-AI/Wan2.2-TI2V-5B-Diffusers fixtures).
